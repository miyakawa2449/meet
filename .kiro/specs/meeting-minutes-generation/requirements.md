# 要件定義書

## はじめに

この機能は、既存の会議話者分離パイプラインに自動議事録生成機能を追加します。システムはPhase 1-5から出力された文字起こしデータ（JSON形式）を処理し、OpenAIのGPTモデルを使用して、要約、決定事項、アクションアイテム、トピック別要約を含む構造化された議事録を生成します。

## 用語集

- **Meeting_JSON**: Phase 1-5から出力される、話者ラベル付き文字起こしデータを含む構造化JSON
- **Minutes_Generator**: Meeting_JSONから議事録を生成するモジュール
- **OpenAI_API**: 自然言語処理に使用するOpenAI APIサービス
- **Action_Item**: 特定の人に割り当てられたタスク（期限はオプション）
- **Decision**: 会議中に到達した結論または決定事項
- **Summary**: 会議内容または特定トピックの簡潔な概要
- **Markdown_Minutes**: 人間が読みやすいMarkdown形式の議事録
- **Structured_Minutes**: 機械可読なJSON形式の議事録
- **Pipeline**: 既存のmeeting_pipeline.pyオーケストレーションシステム
- **CLI**: ユーザーとの対話のためのコマンドラインインターフェース

## 要件

### 要件1: OpenAI API統合

**ユーザーストーリー:** 開発者として、OpenAI APIを統合したい。文字起こしの自然言語処理にGPTモデルを使用できるようにするため。

#### 受け入れ基準

1. Minutes_Generatorは.envファイルからOPENAI_API_KEYを読み込むこと
2. OPENAI_API_KEYが見つからないか無効な場合、Minutes_Generatorは説明的なエラーメッセージを返すこと
3. Minutes_GeneratorはGPT-4とGPT-3.5-turboの両方のモデルをサポートすること
4. OpenAI API呼び出しが失敗した場合、Minutes_Generatorは指数バックオフで最大3回リトライすること
5. すべてのリトライが失敗した場合、Minutes_Generatorは最後の失敗理由を含むエラーを返すこと

### 要件2: Meeting JSON入力処理

**ユーザーストーリー:** ユーザーとして、Meeting JSONを入力として提供したい。システムが文字起こしデータから議事録を生成できるようにするため。

#### 受け入れ基準

1. Minutes_GeneratorはMeeting_JSONへのファイルパスを入力として受け取ること
2. Meeting_JSONファイルが存在しない場合、Minutes_Generatorはファイル未検出エラーを返すこと
3. Meeting_JSONが無効または不正な形式の場合、Minutes_Generatorは説明的なパースエラーを返すこと
4. Minutes_GeneratorはMeeting_JSONセグメントから話者ラベル、タイムスタンプ、テキストを抽出すること
5. Minutes_Generatorは元の会議メタデータ（タイトル、日付、時間）を保持すること

### 要件3: 会議要約生成

**ユーザーストーリー:** ユーザーとして、会議全体の要約が欲しい。会議の内容を素早く理解できるようにするため。

#### 受け入れ基準

1. Minutes_Generatorは会議の簡潔な全体要約を生成すること（100-300語）
2. 要約は議論された主要トピックを特定すること
3. 要約は主要参加者に言及すること（話者ラベルで）
4. 会議時間が2時間を超える場合、Minutes_Generatorはより長い要約を生成すること（300-500語）
5. 要約は入力文字起こしと同じ言語で書かれること

### 要件4: 決定事項の抽出

**ユーザーストーリー:** ユーザーとして、会議中に行われた決定を特定したい。何が合意されたかを追跡できるようにするため。

#### 受け入れ基準

1. Minutes_Generatorは会議中に行われたすべての決定を抽出すること
2. 各決定について、Minutes_Generatorは決定テキストを記録すること
3. 各決定について、Minutes_Generatorはおおよそのタイムスタンプを記録すること
4. 各決定について、Minutes_Generatorは決定を行ったまたは確認した話者を特定すること
5. 決定が検出されない場合、Minutes_Generatorは空の決定リストを返すこと

### 要件5: アクションアイテムの抽出

**ユーザーストーリー:** ユーザーとして、アクションアイテムを特定したい。誰が何をいつまでにする必要があるかを追跡できるようにするため。

#### 受け入れ基準

1. Minutes_Generatorは会議からすべてのアクションアイテムを抽出すること
2. 各アクションアイテムについて、Minutes_Generatorはタスクの説明を記録すること
3. 各アクションアイテムについて、Minutes_Generatorは担当者を特定すること（話者ラベルまたは名前）
4. 各アクションアイテムについて、Minutes_Generatorは言及された場合は期限を抽出すること
5. 各アクションアイテムについて、Minutes_Generatorはおおよそのタイムスタンプを記録すること
6. アクションアイテムが検出されない場合、Minutes_Generatorは空のアクションアイテムリストを返すこと

### 要件6: トピック別セグメンテーション

**ユーザーストーリー:** ユーザーとして、会議をトピックごとに分割したい。特定の議論エリアにナビゲートできるようにするため。

#### 受け入れ基準

1. Minutes_Generatorは会議で議論された個別のトピックを特定すること
2. 各トピックについて、Minutes_Generatorはトピックタイトルを生成すること
3. 各トピックについて、Minutes_Generatorは簡潔な要約を提供すること（50-150語）
4. 各トピックについて、Minutes_Generatorは開始と終了のタイムスタンプを記録すること
5. Minutes_Generatorは会議ごとに最低1つ、最大10のトピックを生成すること

### 要件7: Markdown出力生成

**ユーザーストーリー:** ユーザーとして、Markdown形式の議事録が欲しい。簡単に読んで共有できるようにするため。

#### 受け入れ基準

1. Minutes_Generatorは議事録を含むMarkdownファイルを生成すること
2. Markdown議事録は会議メタデータ（タイトル、日付、時間、参加者）を含むヘッダーを含むこと
3. Markdown議事録は全体要約セクションを含むこと
4. Markdown議事録は番号付きアイテムの決定事項セクションを含むこと
5. Markdown議事録はテーブル形式のアクションアイテムセクションを含むこと（タスク、担当者、期限、タイムスタンプ）
6. Markdown議事録は各トピックのサブセクションを持つトピックセクションを含むこと
7. Markdown議事録は適切なMarkdown書式を使用すること（見出し、リスト、テーブル、タイムスタンプ）

### 要件8: JSON出力生成

**ユーザーストーリー:** 開発者として、JSON形式の議事録が欲しい。構造化データをプログラム的に処理できるようにするため。

#### 受け入れ基準

1. Minutes_Generatorは構造化された議事録を含むJSONファイルを生成すること
2. 構造化議事録はMarkdown出力のすべてのフィールドを機械可読形式で含むこと
3. 構造化議事録は型付きフィールドを持つ一貫したスキーマを使用すること
4. 構造化議事録はエラーなくパースできる有効なJSONであること
5. すべての有効な構造化議事録について、JSONをパースすると同等のデータ構造が生成されること（ラウンドトリップ特性）

### 要件9: CLI統合

**ユーザーストーリー:** ユーザーとして、コマンドラインオプションで議事録を生成したい。ワークフローに統合できるようにするため。

#### 受け入れ基準

1. CLIは議事録生成を有効にする--generate-minutesフラグを受け付けること
2. --generate-minutesが提供された場合、PipelineはMeeting_JSON作成後に議事録を生成すること
3. CLIはGPTモデルを指定する--minutes-modelオプションを受け付けること（デフォルト: gpt-3.5-turbo）
4. CLIは出力言語を指定する--minutes-languageオプションを受け付けること（デフォルト: 入力から自動検出）
5. 議事録生成が失敗した場合、Pipelineはエラーをログに記録するが、Meeting_JSONは保存すること

### 要件10: 出力ファイル命名

**ユーザーストーリー:** ユーザーとして、一貫したファイル命名が欲しい。関連する出力を簡単に識別できるようにするため。

#### 受け入れ基準

1. 入力ファイルが{basename}.mp4という名前の場合、Minutes_Generatorは{basename}_minutes.mdを作成すること
2. 入力ファイルが{basename}.mp4という名前の場合、Minutes_Generatorは{basename}_minutes.jsonを作成すること
3. Minutes_GeneratorはMeeting_JSONと同じディレクトリに出力ファイルを保存すること
4. 出力ファイルが既に存在する場合、Minutes_Generatorは警告ログとともにそれらを上書きすること

### 要件11: エラーハンドリングとロギング

**ユーザーストーリー:** ユーザーとして、明確なエラーメッセージが欲しい。問題を素早くトラブルシュートできるようにするため。

#### 受け入れ基準

1. 議事録生成中にエラーが発生した場合、Minutes_Generatorはコンテキスト付きでエラーをログに記録すること
2. Minutes_GeneratorはAPIエラー、パースエラー、ファイルI/Oエラーを区別すること
3. OpenAI APIのレート制限を超えた場合、Minutes_Generatorは特定のレート制限エラーを返すこと
4. Minutes_Generatorはタイミング情報とともに議事録生成の開始と完了をログに記録すること
5. 議事録生成がスキップされた場合（--generate-minutesフラグなし）、Pipelineはこの決定をログに記録すること

### 要件12: パフォーマンスと効率

**ユーザーストーリー:** ユーザーとして、議事録生成が妥当な時間で完了することを望む。長時間待たないようにするため。

#### 受け入れ基準

1. Minutes_Generatorは1時間の会議の処理を60秒以内に完了すること
2. Minutes_Generatorは利用可能な場合、レイテンシを削減するためにストリーミングAPI呼び出しを使用すること
3. Minutes_Generatorは文字起こしがモデルのトークン制限を超える場合、チャンクで処理すること
4. チャンク化が必要な場合、Minutes_Generatorは結果を一貫して統合すること
5. Minutes_Generatorは議事録生成にかかった時間をログに記録すること

### 要件13: 設定とカスタマイズ

**ユーザーストーリー:** 開発者として、設定可能なプロンプトが欲しい。議事録生成の動作をカスタマイズできるようにするため。

#### 受け入れ基準

1. Minutes_GeneratorはGPTモデル用の設定可能なシステムプロンプトを使用すること
2. Minutes_Generatorはカスタムtemperature設定をサポートすること（デフォルト: 一貫性のため0.3）
3. Minutes_Generatorはカスタムmax_tokens設定をサポートすること（デフォルト: 4000）
4. カスタムプロンプトが提供された場合、Minutes_Generatorはデフォルトの代わりにそれらを使用すること
5. Minutes_GeneratorはAPI呼び出し前に設定パラメータを検証すること

### 要件14: ユニットテスト

**ユーザーストーリー:** 開発者として、包括的なユニットテストが欲しい。機能が正しく動作することを保証できるようにするため。

#### 受け入れ基準

1. テストスイートはOpenAI API統合のテストを含むこと（モック使用）
2. テストスイートはMeeting JSONパースのテストを含むこと
3. テストスイートはMarkdown生成のテストを含むこと
4. テストスイートはJSON生成とラウンドトリップ検証のテストを含むこと
5. テストスイートはエラーハンドリングシナリオのテストを含むこと
6. テストスイートはCLIオプションパースのテストを含むこと
7. 機能が完了したと見なされる前に、すべてのテストが合格すること

### 要件15: ドキュメント

**ユーザーストーリー:** ユーザーとして、明確なドキュメントが欲しい。議事録生成機能の使い方を理解できるようにするため。

#### 受け入れ基準

1. README.mdは使用例を含む議事録生成のセクションを含むこと
2. ドキュメントは--generate-minutes CLIオプションを説明すること
3. ドキュメントは生成されたMarkdownとJSON出力の例を提供すること
4. ドキュメントはOPENAI_API_KEYのセットアッププロセスを文書化すること
5. ドキュメントは一般的な問題のトラブルシューティングのヒントを含むこと
